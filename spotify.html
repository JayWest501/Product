Prioritization
🎯
Daily Mix & Discover Weekly
Genuinely best-in-class personalization. Low effort, high perceived value, and delivered on a predictable schedule users build habits around. This is the benchmark for music app personalization.
🗺️
Made For You hub
Discovery has a permanent, predictable home instead of being buried in the feed. Users know where to go. Sounds obvious — most apps still don't do this well.
📱
Cross-device session handoff
Nearly invisible. Close the laptop, open your phone, it's still playing. A real engineering and product win that users have never consciously noticed — which is exactly the mark of excellent UX.
01
Search intent is ambiguous
When a user searches "true crime," they could mean a podcast, an audiobook, or a playlist. Spotify guesses, mixing all three into one results feed. That guess is wrong often enough that users develop workarounds — typing more specific queries to force a result type. That's the user doing work the product should be doing.
02
The home screen is unstable
The layout reshuffles constantly based on algorithm changes. Users lose the ability to build a mental map of where things live. A home screen's job is to be predictably navigable. When it moves too often it stops being navigation and starts being discovery — whether the user wanted discovery or not.
03
Algorithmic playlists lack explainability
When Discover Weekly surfaces a track, there's no signal about why. Users who love the pick have no way to signal "more like this" beyond a save. Users who want to understand the pattern can't. This limits trust and discovery confidence over time.

The insight
The filter persists as a user preference — not reset every session. Set it once, applied everywhere. One upfront decision eliminates friction on every subsequent search. That compounds across millions of searches per day.
Persistent content-type filter
Set once, applied everywhere. Not re-asked on every search. Music / Podcasts / Audiobooks as a user preference, not a query modifier. Reduces cognitive load on every subsequent search session.
"Why this track" tag
A lightweight explainability label on algorithmic picks — similar to what TikTok does with its For You signals. "Based on: The National, songs you saved this week." Low build cost, meaningfully higher trust.
Pinned home layout option
An opt-in stable layout for power users who want predictability over surprise. The algorithm still runs but the furniture stops moving. Targets a real segment that currently churns to competitors.
The through-line across all three: reduce the work the user has to do. If a user is compensating for something the interface should handle, that's a product problem worth fixing.

How I prioritize
I ask: does this fix something the user is currently working around? The filter and explainability tag both do. The layout option is real but addresses a smaller specific segment — worth validating demand before committing build time.
